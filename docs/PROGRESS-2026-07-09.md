## Progress summary

### 1. Gravity Forms localization (standalone pages)
- Audited every form: contact section + most pages = **JotForm** (per-locale, already
  localized); **Careers / Support / Get A Quote** pages = **Gravity Forms**.
- Created **11 translated GF copies** (FR/ES/PL) of #2 Careers, #4 Support, #5 Get A
  Quote (labels/choices/button/confirmation translated).
- Wired via **per-page `form_id`**: Support/Quote templates already read it; changed
  `wardjet-careers.php` from hardcoded `gravity_form(2)` to `get_field('form_id') ?: 2`
  with a dynamic privacy-label selector. Verified per-locale render.
- Confirmed the JotForm contact section is per-locale already; reverted an initial
  wrong attempt to convert it to Gravity.

### 2. Contact section — right column (HQ location card + Global Presence)
- Root cause of the empty right column: the `contact_localized` repeater had no rows.
- Registered the base repeater + injected Location Card / Global Presence / CTA
  sub-fields in `inc/acf-contact-extra.php`; ran injected fields through
  `acf_get_valid_field()` to clear ACF notices.
- Populated 6 per-locale rows with **real WardJet HQs** (NA/Tallmadge, UK/Telford,
  Poland/Katowice) + translated Global-Presence copy/regions.
- Fixed address line-spacing (field `new_lines` vs the `white-space: pre-line` CSS).

### 3. "Get a Quote" CTA + old-site links
- Header CTA pointed at the **old live site**; added `lc_resolve_link_for_locale()` in
  `header.php` so same-site CTAs resolve to the current locale's translated page.
  Get-a-Quote now localizes per locale (correct slugs + labels).
- Rewrote every `wardjet.com` link to the current host (menus, products mega-menu ACF,
  company-card CTA, About template), leaving genuine externals (clients.wardjet.com).
- Removed the dead commented `gravity_form(3)` block in `wardjet-contact.php`.

### 4. Per-locale listing filters
- `wardjet-our-products.php` + `wardjet-industries.php`: series/industry CPT queries now
  filter by `region_language_code` (en-us fallback for en-ca/en-uk). Each locale shows
  its own products/industries with localized titles.
- **Accessories (es)** page rendered blank: flexible-content layout keys were translated
  (`banner_principal`/`galería_de_imágenes`) → remapped to canonical keys; data was
  already fully translated. Isolated to one page (scanned; not systemic).

### 5. Videos pages (all locales)
- `section-videos_gallery.php`: the `section` taxonomy has a **separate term per locale**;
  pages stored the English term IDs → every locale showed English videos. Now resolves
  selected sections by **name** and filters videos by `region_language_code` (tabs +
  video CSS classes aligned via `sanitize_title`). Each locale shows its own videos.
- Fixed **404s** from draft/publish slug collisions (es + fr videos pages) via a
  systemic per-locale draft-slug rename + rewrite flush. (PL has 0 videos — content gap.)

### 6. Cross-locale menu alignment (EN as template)
Rebuilt the ES/FR/PL menus to match the EN structure (order, items, submenus) with
per-locale translations + localized links (auto-resolved through `translation_group_id`;
dictionary for custom items):
- **Upper nav:** Buy Parts, Resources ▾ (News/Blogs/Webinars/Brochures), Locations.
- **Main nav:** Products, Industries, Features, Software, Support, About Us + all submenus.
- **Footer-main:** About Us, Industries, Products, Options, Software, Support, Companies.
- **Footer-nav:** Terms & Conditions, Privacy Policy.
- Footer **Careers** repointed from the AXYZ UltiPro portal to each locale's careers page.
- en-us/en-ca/en-uk share the EN menus (reference) — already consistent.
